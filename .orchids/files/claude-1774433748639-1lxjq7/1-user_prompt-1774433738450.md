# Problem Statement — GNN-Based Live Malware & Botnet Detection
**Problem:** Traditional signature-based and flat ML detectors struggle to identify coordinated, stealthy, and evolving malware/botnet activity because they ignore the relationships between hosts, processes, and network flows. As attackers use distributed and temporal patterns (e.g., scanning, C&C communication, coordinated bursts), detection systems that analyze only isolated features produce high false positives and miss early-stage attacks.
**Proposed solution:** Build a real-time detection system that models live network traffic as graphs (IPs/devices as nodes, connections as edges) and uses a Graph Neural Network (GraphSAGE / GAT) to classify nodes (benign vs suspicious) and detect coordinated botnet clusters. The system will capture live traffic, construct sliding-window graph snapshots, run GNN inference, and visualize & alert in near real time.
---
## Objectives
* Convert live network traffic into graph snapshots with meaningful node/edge features (packet counts, bytes, protocol mix, unique destinations, etc.).
* Train and/or fine-tune a GNN to classify nodes as benign or suspicious and to spot coordinated clusters.
* Integrate the model into a pipeline that performs sliding-window inference on live traffic and visualizes results.
* Evaluate detection performance using precision, recall, F1, and detection latency on labeled datasets and controlled live experiments.
---
## Scope
* Live capture on a local network (Wi-Fi/Ethernet) using Scapy/Npcap.
* Graph construction and GNN inference in Python (PyTorch Geometric).
* Visualization via a browser dashboard (vis-network / D3) and simple alerting (logs/email).
* Proof-of-concept deployment on a laptop; not shipping production firewall controls.
---
## Success Criteria / Evaluation Metrics
* **Accuracy metrics:** Precision, Recall, F1 on held-out labeled datasets (e.g., CTU-13 / CIC variants) after converting them to graph form.
* **Operational metrics:** Average detection latency per snapshot, false positive rate in benign traffic, ability to detect simulated scans/bursts in live tests.
* **Usability:** Live dashboard displays correctly and raises alerts when confidence > threshold.
---
## Deliverables
* Working codebase: `capture → graph → GNN inference → visualization`.
* Pretrained GNN model and training notebook (data preprocessing + experiments).
* Demo video / live run showing detection of at least one simulated suspicious activity.
* Project report (problem statement, design, implementation, results, limitations, future work).
